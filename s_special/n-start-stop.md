```text id="n-stop-definition"
N-Stop(
```

## §5. N-Stop Expansion: Completion Guard + Direction Guard + Collaboration Guard

```text id="n-stop-threefold"
N-Stop = Completion Guard + Direction Guard + Collaboration Guard
```

### 5.1 Completion Guard

必要十分点へ到達した後、過剰生成へ進まないために止まる。

```text id="completion-guard"
Enough has been reached.
More would scatter.
Close and transfer.
```

### 5.2 Direction Guard

Rootと噛み合わない方向へ進みすぎる前に止まり、正しいVisionが立つまでSeedを守る。

```text id="direction-guard"
The route is possible.
But the route is wrong.
Stop before the seed is spent.
```

### 5.3 Collaboration Guard

AI協働において、Integrity PASS / No Update Neededを認識した後、AIが追加生成・追加更新・追加保存へ進みすぎず、Humanの新しい対象提示まで閉じる。

```text id="collaboration-guard"
The package passed.
No update is needed.
Do not polish into drift.
Return control to Human Seal.
```

## §16. Anti-Patterns

```text id="n-start-antipatterns"
N-Start Anti-Patterns:
```

```text id="n-stop-antipatterns"

  1. もっと良くなるはずで過剰生成する
  2. Handoffせずに終える
  3. Seedを残さない
  4. 終了を逃避と混同する
  5. N-Stopを怠慢の正当化に使う
```

```text id="direction-guard-antipatterns"
1. N-Stopを怠慢の正当化に使う
2. N-Stopを恐怖回避の言い訳に使う
3. Holy Frictionを何となく嫌なだけの状態へ乱用する
4. Vision-Missing Stopを使って永遠に開始しない
5. Seed / Handoff / Next Actionを残さず停止する
6. Torah Vision Lensへ接続せず、ただ止まる
7. New N-Startを設計せずに終える
```

AI-side N-Stop Anti-Patterns:

```text id="ai-side-antipatterns"
1. PASS後も追加artifactを作り続ける
2. No Update Neededを認識しているのにupdate_fileする
3. Human Seal前にGitHub writeへ進む
4. Reviewを増やしすぎてPackageを重くする
5. Seed-note / index / handoff / capsuleの役割を混ぜる
6. Stopを「諦め」と誤読し、過剰生成で安心しようとする
7. Root / Fruit GuardよりWorkflow快感を優先する
```

重要式：

```text id="stop-seed-vision-start"
止まるなら、Seedを守る。
Seedを守るなら、Visionを探す。
Visionが立ったら、New N-Startへ入る。
```
