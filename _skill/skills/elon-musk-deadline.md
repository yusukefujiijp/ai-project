## 7. Output Ladder / 出力形式

締切時点で品質が足りない場合、締切を消さない。  
Output Modeを変える。

これはElon Musk Deadlineを安全化する中核である。  
締切を守るとは、常にPaid品質で出すという意味ではない。  
締切を守るとは、到達品質に応じた形式でOutputをSealすることである。

### Programming-Like Block

```yaml
output_ladder:
  rule:
    - "Do not delete the deadline."
    - "Change output mode."
    - "Never fake quality."

  levels:
    1_final_product:
      meaning: "完成品として出す"
      use_when:
        - "quality is sufficient"
        - "integrity guard is clear"

    2_paid_article:
      meaning: "有料記事 / 有料商品として出す"
      use_when:
        - "buyer value is clear"
        - "promise is honest"

    3_free_public_note:
      meaning: "無料公開メモとして出す"
      use_when:
        - "public value exists"
        - "paid quality is not reached"

    4_prototype:
      meaning: "試作品として出す"
      use_when:
        - "structure is valuable"
        - "completion is partial"

    5_internal_harvest:
      meaning: "内部Harvestとして保存する"
      use_when:
        - "public release is premature"
        - "learning should be preserved"

    6_failure_report:
      meaning: "失敗・未達・詰まりをReportとしてSealする"
      use_when:
        - "deadline was not met"
        - "cause and patch can be named"

    7_next_deadline_patch:
      meaning: "次回締切へのPatchとして渡す"
      use_when:
        - "next cycle needs improvement"

  compression:
    - "品質不足は、締切消滅の理由ではない。"
    - "品質不足は、Output Modeを変更する理由である。"
```
