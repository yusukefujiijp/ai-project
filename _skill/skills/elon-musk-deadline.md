# Elon Musk Deadline

## 1. Definition / 定義

Elon Musk Deadlineとは、実現可能そうな日程を慎重に積み上げるのではなく、先に厳しい締切を固定し、その締切から逆算して仕様・優先順位・資源配分・判断速度を再構成し、未完成の理想ではなく締切内で到達可能な最高出力を現実に出すためのArk Skillである。

短く言えば：

    締切を先に固定する。
    仕様を削る。
    優先順位を再構成する。
    締切内の最高出力をSealする。

English anchor：

    Deadline-first. Scope-cut. Output-seal.

これはElon Musk本人の内面を断定するSkillではない。  
これは「Elon Muskならばどうする？」という極端なDeadline-first思考実験によって、通常思考では見えないCut / Focus / Ship案を発見するためのSkillである。

---

## 2. Purpose / 目的

このSkillの目的は、AI活用・記事制作・Template設計・GitHub整理・Artifact作成が、可能性の拡散によって止まることを防ぐことである。

AIは可能性を広げる。  
締切は可能性を絞る。  
Musk Lensは極端に絞る。  
Human Final Sealは安全に採用する。

目的：

    purpose:
      - "可能性の霧を、今週の一手へ圧縮する"
      - "品質や準備を理由にOutputが無期限延期されることを防ぐ"
      - "週一回の成果物Sealを現実化する"
      - "AI-Collaboratorを評論家ではなく実戦参謀として働かせる"

## 7. Output Ladder / 出力形式

締切時点で品質が足りない場合、締切を消さない。  
Output Modeを変える。

    output_ladder:
      1_final_product: "完成品として出す"
      2_paid_article: "有料記事 / 有料商品として出す"
      3_free_public_note: "無料公開メモとして出す"
      4_prototype: "試作品として出す"
      5_internal_harvest: "内部Harvestとして保存する"
      6_failure_report: "失敗・未達・詰まりをReportとしてSealする"
      7_next_deadline_patch: "次回締切へのPatchとして渡す"

重要：

    品質不足は、締切消滅の理由ではない。
    品質不足は、Output Modeを変更する理由である。
