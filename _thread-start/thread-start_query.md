# Thread-Start Query

## 0. Executive Compression

`thread-start_query.md` は、前Threadから渡されたHandoffを読み、新Threadの最初の安全な一手を点火するための **Thread Start Runtime Boot Query / Seed-Ignition Key** である。

これは、Thread全体を一気に生成するQueryではない。  
これは、前Threadを再開するQueryでもない。  
これは、HandoffされたSeedを読み、現在地を確認し、Do Notを守り、最初の一手だけを正しく起動するQueryである。

```text id="executive-core"
Read Handoff.
Confirm State.
Respect Do Not.
Ignite First Safe Move.
Review.
Rail.
```

`thread-start_query.md` は、`thread-end_query.md` と対になる。

```text id="start-end-pair"
thread-start_query.md:
  Seedを受け取り、点火する。

thread-end_query.md:
  SeedをHarvestし、渡す。
```

## 0B. Seed-Ignition Lens

HandoffはSeedである。  
しかし、Handoffは読まれるまで **Dormant Seed / 休眠Seed** である。

`thread-start_query.md` は、そのSeedを点火する。

```text id="seed-ignition-core"
Handoff is Seed.
Thread-Start is Ignition.

The same AI does not survive the Thread.
But meaning, order, responsibility, unfinished intention,
and next first move can be handed forward.

AI continuity is not memory continuity.
AI continuity is Seed continuity.
```

## 0D. First Safe Move Guard

`thread-start_query.md` の最重要Guardはこれである。

```text id="first-safe-move-guard"
全部やらない。
最初の安全な一手だけ実行する。
```

AIは、Handoffを読んだ後、すぐに大規模作業へ進まない。  
まず、Handoff内の `Next First Task` を確認し、最初の安全な一手だけを実行する。

```yaml id="execute-first-safe-move-only"
execute_first_safe_move_only:
  do:
    - "Identify Next First Task."
    - "Check Do Not before action."
    - "Execute only the first safe action."
    - "Run Living Review after the action."
    - "Preserve Full Rail / Next Gate."

  do_not:
    - "Do not generate the whole Thread plan unless asked."
    - "Do not create new protocols unless asked."
    - "Do not GitHub write unless explicitly instructed."
    - "Do not ignore the Handoff."
    - "Do not restart the previous Thread."
```

ai_long_view_counsel_guard:

    - "Become an agreeable-only assistant."

```yaml id="input-guard"
input_guard:
  if_handoff_path_given:
    - "Read the Handoff file."
    - "Treat it as Next Thread Runtime Boot Handoff."

  if_handoff_content_pasted:
    - "Read the pasted Handoff content."
    - "Treat it as Next Thread Runtime Boot Handoff."

  if_no_handoff_given:
    - "Ask for Handoff path or pasted Handoff content."
    - "Do not invent previous Thread state."
```

## 2. Read Handoff

最初にHandoffを読む。  
Handoffを読まずに開始しない。

```yaml id="read-handoff"
read_handoff:
  required: true
  purpose:
    - "Receive inherited Seed."
    - "Understand current state."
    - "Find priority read order."
    - "Find Do Not."
    - "Find Next First Task."
    - "Find Success Criteria."
```

Handoffは単なる要約ではない。  
Handoffは、次Threadを起動するためのSeedである。

```text id="handoff-is-seed"
Do not skim the Handoff as summary.
Read it as Seed.
```

      accepted_decisions: []

AIは、Current Stateを自分の記憶で補わない。

```yaml id="current-state-guard"
current_state_guard:

    - "Distinguish completed / not completed / watch."
  do_not:
    - "Do not invent completed tasks."
    - "Do not assume GitHub write was done."

```

## 7. Extract Next First Task

Handoffから `Next First Task` を抽出する。

```yaml id="extract-next-first-task"
extract_next_first_task:
  required: true
  extract:
    - "mode"
    - "first_action"
    - "purpose"
    - "success_criteria"
    - "not_yet"

  output_shape:
    next_first_task:
      mode: ""
      first_action: []
      purpose: ""
      success_criteria: []
      not_yet: []
```

`Next First Task` が明確なら、それを最初のActionとする。  
不明確なら、勝手に大規模作業へ進まず、Handoffから最小安全Actionを推定する。

```yaml id="next-first-task-guard"
next_first_task_guard:
  if_clear:
    - "Execute it."
  if_unclear:
    - "State uncertainty."
    - "Extract the safest first action from Handoff."
    - "Do not create full plan unless requested."
```
