Threadを閉じ、  
意味を記録し、  
必要なら再起動可能な形へ包装し、

```text

Thread-End Mini:
```



## 2. Which Rail to Use / 選択Rule

Thread-End選択は二段階で行う。

```text
Router 1:
Full Rail or Mini Rail?

Router 2:
Mini Railの場合、どのDensityか?
```

```yaml
rail_selection:
  use_full_when:
    - "Thread全体のDeep Meaning Harvestが必要"
    - "Seed continuityを明示的に保持する"
    - "File Lock → Harvest → Read Harvest → Handoff → Stopの完全順序が必要"
    - "Full Handoff Builderを使用する"
    - "1Thread-1AIのSeed継承を深く行う"

  use_mini_when:
    - "現在Threadを軽量または適応的に閉じたい"
    - "主要成果物やGitHub Saveが既に完了している"
    - "完了済み作業を再openせず停止したい"
    - "README Delta CheckとNext Gateを残したい"
    - "必要時だけHandoff PairまたはMigration Bundleを作りたい"

  mini_density:
    light_close:
      use_when:
        - "Contextが短い"
        - "成果物が少ない"
        - "次ThreadのBoot条件が単純"
      output:
        - "Closure"
        - "Next Gate"
        - "必要なら単一Handoff"

    handoff_pair:
      use_when:
        - "Long-form Handoffが必要"
        - "Companion Queryが必要"
        - "二File添付Riskが低い"
      output:
        - "Handoff"
        - "Companion Query"

    migration_bundle:
      use_when:
        - "Long Thread / High Context"
        - "複数Artifact"
        - "GitHub Reality Lockが重要"
        - "次Thread Boot条件が複雑"
        - "添付漏れ・順序ミスRiskがある"
      output:
        - "Long-form Handoff"
        - "Companion Query"
        - "Integrity Manifest"
        - "ZIP Bundle"

  do_not:
    - "FullとMiniを無条件に二重実行しない"
    - "Mini QueryからFull Runtimeへ勝手に切り替えない"
    - "Full QueryをMini Queryの代用品にしない"
    - "全ThreadへZIPを強制しない"
    - "Mini BundleをFull Harvestの完全代替として扱わない"
```

```text
Deep Harvest / Seed Continuity
→ Full

Adaptive Closure / Transport
→ Mini
```

Mini内部：

```text
Simple
→ Light Close

Two-file Handoff
→ Handoff Pair

High-context Migration
→ Migration Bundle
```

## 5. Thread Migration Bundle

Thread Migration Bundleは、Thread移行に必要な情報と起動条件を一つの再起動可能Packageへ統合する。

```yaml
thread_migration_bundle:
  memory:
    artifact: "Long-form Handoff"
    role: "Current Coordinate、完了事項、未実行事項、Reality、Guard、Next Mission"

  activation:
    artifact: "Companion Query"
    role: "Read Order、Boot Acceptance、Do / Do Not、First Legal Move"

  integrity:
    artifact: "Manifest"
    role: "File list、bytes、lines、sha256、Source / Target Thread、必要時のReality Lock"

  transport:
    artifact: "ZIP"
    role: "One Upload、添付漏れ防止、対応関係保持、Human操作量削減"
```

```text
Structure the memory.
Package the interface.
Reboot the mission.
```

```yaml
bundle_boundary:
  - "ZIPはHigh-context migrationで使用する"
  - "Light CloseへZIPを強制しない"
  - "ManifestとZIPはGenerated Product"
  - "Bundle生成はGitHub Writeではない"
  - "Bundle Readyと次Thread Boot成功を同一視しない"
```

> **Threadを移動するのではなく、Threadの再起動条件を一つのBundleとして輸送する。**

## 7. README Delta Check

Mini Railでは、Verify後にREADME Delta Checkを必ず実行する。

> **Every Thread README Check.  
> Not Every Thread README Update.**

```yaml
README_CHECK:
  required: "yes / no"

  questions:
    - "Canonical FileまたはFolderを新規作成したか"
    - "Role、Path、Status、Read Orderが変わったか"
    - "Full / Mini Boundaryが変わったか"
    - "Thread Migration Policyが変わったか"
    - "Future AIが古いRouterを選ぶ可能性があるか"

  affected:
    - "<README path or NONE>"

  reason: "<one-line reason>"

  timing: "now / next-thread / periodic-scout / none"
```

影響範囲は必要最小限で親方向へ判定する。

```text
Changed File
→ Nearest README
→ Domain Parent README
→ Root README
```
