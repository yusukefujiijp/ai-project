# Ark99

## 0. Ark99とは

Ark99は、Ark Projectにおける **Free Spawn / Wild Seed Field** である。

つまり、まだ正式Project化しないSeed・Naming候補・小さな成功・未成熟なPatternを、重い構造に押し込めず、public-safeに観測するための入口である。

```text
Ark99:
  早すぎるProject化を避ける。
  しかし、小さなSeedを消さない。
```

Ark99は最終Sealではない。  
Ark99は自動Project化装置ではない。  
Ark99は、Seedが次のThreadやReality Responseで本当に生き残るかを見るための軽いFieldである。

---

## 1. Current GitHub Phase / 現在のGitHub段階

このFolderは、**README-first GitHub Bet** の一部である。

現在のScopeは次の通り。

```text
現在やること:
  public-safeなArk99 READMEを置く。

現在やらないこと:
  raw thread logsを置かない。
  private personal recordsを置かない。
  Mission Card / S/G/runtime/tasksを一括移行しない。
```

GitHub化は、English-first化ではない。  
GitHub化は、public-safeな正準記録をstable pathへ置き、Git履歴でVersionを保つためのCanonical化である。

```text
GitHub = canonical public-safe record.
Git commit history = version ledger.
```

---

## 2. Ark99の役割: Wild Seed Field

Ark99は、まだ分類しきれないSeedを受ける。

ここで受けるものは、たとえば次のようなもの。

```text
- early signals
- naming candidates
- lightweight concepts
- unexpected success
- reusable workflow patterns
- public-safe project entry notes
```

日本語で言えば、Ark99は「まだ名前が固まっていないが、消すには惜しいもの」を受ける場所である。

ただし、Ark99に置かれたからといって、すぐ正式Projectになるわけではない。

```text
Ark99 receives.
Ark99 observes.
Ark99 does not automatically promote.
```

---

## 3. Second-Thread Gate

Ark99の重要Signalは、**Second-Thread Gate** である。

同じSeed・Theme・Named Patternが第二Threadにも現れた場合、それはArk00 pickup candidateになる。

ただし、これは即昇格ではない。

```text
一度だけ出たSeed:
  Ark99で観測する。

第二Threadでも生き残ったSeed:
  Ark00 pickup候補としてReviewする。

現実運用で繰り返し働くSeed:
  selected canonical slotへの候補になる。
```

Second-Thread Gateは、早すぎる保存と早すぎる拡張を避けるためのGateである。

## 4. Signal / Skill Direction

Ark99は、Signalを観測する。

特に見るべきものは次である。

```text
- repeated themes
- unexpected success
- reusable naming
- useful workflow patterns
- human-AI semi-automation loops
```

Signalは、すぐSkillではない。  
しかし、繰り返し現れ、実運用で働き、Future AIが再起動できる形に整えば、Skill候補になる。

```text

  現れた兆し。

Skill:
  再利用できる形まで育った働き。
```

Ark99は、この変化を急がず観測する。

## 7. Workflow Signal: Create-and-Patch Rail

有用なWorkflow Signalが現れた。

```text
Create-and-Patch Rail:
  新しい正準ファイルを作る。
  Reality Responseが必要を示した時だけ、
  対応する既存SSOTへ小Patchを当てる。
```

これは、Ark-OKF Engine / Ignition Key pairで確認された。

```text
Engine:
  s_special/ark-open-knowledge-format.md

Ignition Key:
  s_special/ark-open-knowledge-format_query.md
```

このRailの意味は、単に二つのファイルを作ることではない。

```text
新規作成:
  必要になった正準ファイルを作る。

小Patch:
  既存SSOT側に、必要な接続だけを追加する。

Guard:
  local Markdown duplicateを原則作らない。
```

これはSignalであり、自動拡張ではない。

```text
No local Markdown duplicate by default.
GitHub stable paths carry the canonical public-safe record.
Git commit history carries the version ledger.
```
