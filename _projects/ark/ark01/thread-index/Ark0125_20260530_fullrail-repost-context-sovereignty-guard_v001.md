## §3. Problem Diagnosed

### §3.1. Bug Pattern

The failure mode in Ark0125 was not a lack of fields. The fields were present.

The bug was a context-layer inversion:

```text
User reposted the FullRail.
AI read the embedded stale Plan flag.
AI treated create_files_now:false as still active.
AI asked User to repair booleans.
Commit rhythm stalled.
```

The AI over-trusted the inner packet text and under-read the outer user action.

### §3.2. Why This Matters

FullRail was designed to reduce roundtrips, not create boolean homework.

If the User must manually repair every stale field before commit, the FullRail has failed its operational purpose.

The repost itself is the higher-order context signal.

## §10. Living Review Judgment

The living judgment is that v036 is a small but decisive runtime patch.

The prior mistake looked minor: AI simply followed `create_files_now:false`. But the deeper issue was role confusion.

The packet text was created by AI as a candidate.
The repost act was performed by the User as a commit signal.

Therefore, the outer act carries newer intent than the inner stale safety flag.

This is the AGI-relevant layer: context hierarchy matters.

A capable future AI must read not only object-level fields but also the speech-act layer:

```text
Who emitted this?
When was this flag created?
Is this flag still active, or is it stale candidate metadata?
What is the outer user act doing now?
```

## §13. Stop Conditions

Stop file creation if:

- placeholder remains in required fields
- target thread coordinate / date / slug cannot be safely resolved
- User explicitly says “まだ作成しない”
- User explicitly says “Plan Mode継続”
- User explicitly says “Review only”
- User explicitly says “observe_only”
- unsafe ambiguity remains
- artifact generation would exceed scoped target

Do not stop merely because:

- the reposted packet still contains `Plan Mode`
- the reposted packet still contains `create_files_now:false`
- the reposted packet still says `not commit`
- smart quotes appear in the packet
