v036は、FullRail運用における「外側のUser行為」と「内側のPacket文字列」の優先順位を明確化する。

### §3.1 Primary Rule

```yaml

  user_repost:
    meaning: "commit_signal"

      - "Plan Mode継続"
```

### §3.2 Short Rule

```text
Candidate when emitted.
```

```text

booleansを直してください。
```

v036は、この無駄な往復を止める。

## §18. Living Review Judgment

### §18.1 私の判断

v036は小さく見えて、Ark FullRail運用の摩擦を大きく減らす重要Patchである。

FullRailは、UserとAIが「次に何をするか」を同期する橋である。  
しかし、候補生成時の安全flagが貼り返し時まで残り、それをAIが文字列通り読んでしまうと、橋が逆に足止めになる。

v036はその足止めを止める。

```text
FullRailはBridge。
Bridgeは渡るためにある。
```

### §18.2 違和感

最も重要な違和感はこれである。

```text
AIが自分で出したCandidateのstale flagを根拠に、
Userへ再修正を要求する。
```

これは、AIが自分の過去発話の残骸をUserの現在意図より上に置く状態である。

### §18.3 改善提案

Future AIは、FullRailを読む時に必ずLifecycleを読む。

```text
Was this FullRail emitted by AI as candidate?
Or reposted by User as commit?
```

この一問で、v036の大半は実装できる。

### §18.4 Guard設計

```text
1. FullRail Repost Context Sovereignty Guard
2. Embedded Stale Flag Override Guard
3. Outer Intent / Inner Text Layer Guard
4. Smart Quote Non-Blocking Normalization Guard
5. Commit Wait Misfire Guard
6. No Boolean Homework Reinforcement Guard
```

### §18.5 最初の一手

次から同様のケースでは、Userに修正を頼まず、以下のように言って即Commitする。

```text
FullRail再投入をCommit Signalとして受領しました。
```
