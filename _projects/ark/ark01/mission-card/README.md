# Ark01 Mission Cards

This folder stores one-by-one crafted Mission Cards for Ark01 threads.

These files are not generic summaries.  
They are thread-specific reboot cards that preserve each thread’s Mission, First Gate, Root Guard, Meta Layer, Living Review, and Misread Prevention.

---

## §0. Purpose

This folder is the Mission Card shelf for Ark01.

Each Mission Card is designed to help a future AI or human reader quickly recover:

- what happened in a specific Ark01 thread,
- what Mission emerged from that thread,
- what First Gate should be opened,
- what must not be misread,
- how the thread returns to 主イェシュア・ハマシア,
- and how the thread can be reused without flattening it into a generic routine.

Mission Cards are not replacements for the source thread analysis files.  
They are distilled, rebootable Mission Gates.

```text id="9se2dw"
Source is the living body.
Craft is the reading method.
Mission Card is the crystallized gate.
Root is 主イェシュア・ハマシア.
```

## §2. Naming Policy

This folder uses simple filenames for daily operation.

Example:

```text id="9bfx8i"
ark0107_mission-card.md
```

Simple filenames are used because they are easier to scan, link, and maintain inside this folder.

However, source identity is still preserved inside each Mission Card’s YAML metadata.

Each Mission Card should preserve:

```yaml id="hgbmo5"
metadata_must_preserve:
  - "target_thread_coordinate"
  - "filename_anchor"
  - "thread_start_date_slot"
  - "thread_start_date_status"
  - "target_thread_semantic_slug"
  - "source_file"
  - "craft_file"
```

```text id="vf1scg"
Simple filename is for operation.
Source identity is preserved inside each Mission Card metadata.
```

## §3. Mission Card Rule

Mission Cards in this folder should follow the Ark Mission Card Craft principle:

```text id="dswk4f"
1 Source.
1 Mission Card.
1 Living Review.
1 Commit after User Final Seal.
```

Each Mission Card should be created one-by-one.

Do not batch-generate Ark01 Mission Cards.  
Do not treat a previous Mission Card as a copy-paste template.  
Do not flatten a thread into a generic productivity routine.

Mission Cards should preserve:

```yaml id="2ntplm"
mission_card_must_include:
  - "Root Guard"
  - "Source Note"
  - "Source Pulse"
  - "Mission Seal"
  - "Thread-specific First Gate"
  - "Meta Layer / Living Review Layer"
  - "Victory Equation"
  - "Misread Prevention"
  - "Future Reuse / Project-specific Translation"
  - "Living Review"
  - "FullRail Closure when relevant"
```
