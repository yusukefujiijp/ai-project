# Ark Project

## 0. Ark Projectとは

Ark Projectは、**Daily Teshuvah Gate-to-Yeshua** をRootに置く、public-safeなHuman-AI collaboration fieldである。

このGitHub上のArk Projectは、Arkのすべてを移す場所ではない。  
まず、public-safeに読める入口・Stage Map・README-first patternを置く場所である。

```text
Ark Project on GitHub:
  public-safeな入口を置く。
  private-depthを置かない。
  GitHub stable pathで正準記録を保つ。
```

GitHub化は、English-first化ではない。  
GitHub化は、public-safeな正準記録をstable pathへ置き、Git履歴でVersionを保つためのCanonical化である。


`ark00/` が開いたことは、full projectizationを意味しない。

## 3. Ark99 / Ark00 / Ark01+ の違い

Ark99、Ark00、Ark01+は似ているが、役割が違う。

```text
Ark99:
  受ける場所。
  Wild Seed Field。

Ark00:
  選び始める場所。
  Selective Incubation / Project Embryo。

Ark01+:
  正式Project段階。
  Formal Project / field-tested project stages。
```

Ark99は、早すぎるProject化を避けながらSeedを消さないためのFieldである。

Ark00は、すべてのSeedを自動昇格させる場所ではない。  
繰り返し現れ、軽く形を持ちはじめたSeedを、正式Project化前に温存・軽量検証する場所である。

Ark01+は、現時点ではfuture directionである。  
Ark00の存在は、Ark01+の自動作成を意味しない。

## 5. AI Scout Pass / Repository Reality Review

Ark Projectでは、重要なREADME更新やGitHub Commit後に **AI Scout Pass** を使う。

```text
AI Scout Pass:
  AI-CollaboratorがRepository上の現実状態を読み、
  README・folder・canonical fileのズレを検出し、
  人間がまだ気付いていない改善点をLiving Reviewとして提案するPass。
```

正式名としては、**Repository-Triggered AI Foresight** と呼べる。

日本語では、**Repository起点AI先取り気付き** である。

今回の重要なUnexpected Successは、AIが親Ark READMEとRoot READMEの仮置き感を先に検出したことである。

```text
Stable GitHub State
+ Current Conversation Intent
+ Existing README Pattern
+ AI Delta Detection
= Proactive Living Review
```

AI Scout Passは、Commitの自動実行ではない。  
それは、先取り気付き → Living Review → Human Seal のためのReview工程である。
