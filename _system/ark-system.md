## 6. Reality Review Protocol

Reality Reviewは、作成したArtifactやGitHub上のFileが、本当に意図通り存在しているか確認するProtocolである。

```yaml id="reality-review-protocol"
reality_review_protocol:
  required_after:
    - "GitHub Direct Create"
    - "GitHub Update"
    - "Download Markdown Generation"
    - "Major Artifact Creation"

  check_items:
    - "target path exists"
    - "frontmatter exists"
    - "canonical_path is correct"
    - "role / not_type are correct"
    - "important Guard sections are present"
    - "GitHub write did not drift"
    - "blob_sha or commit_sha is recorded when available"

  output:
    - "PASS / FAIL"
    - "evidence"
    - "correction_condition"
    - "next_gate"
```

```yaml id="github-direct-write-rule"
github_direct_write_rule:
```
